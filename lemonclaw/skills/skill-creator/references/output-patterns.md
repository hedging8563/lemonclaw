# Output Patterns

Use this file when the skill needs predictable output quality.

## Pattern 1: Ready-to-Run Deliverable

Use when the result should be executed immediately.

Checklist:
- Include the exact command, file, or artifact.
- Keep placeholders explicit.
- Mention the expected success signal.

## Pattern 2: Review-Friendly Summary

Use when another person will inspect the work.

Checklist:
- Lead with the important outcome.
- Group related changes together.
- Call out residual risk and missing validation.

## Pattern 3: Diff-Oriented Explanation

Use when modifying an existing system.

Checklist:
- Explain what changed.
- Explain why it changed.
- Explain what behavior is now different.

## Pattern 4: Structured Decision Output

Use when the skill compares options.

Checklist:
- Present the recommendation first.
- Keep each option's tradeoff on one line.
- End with the deciding factor.

## Pattern 5: Reusable Template

Use when the skill will generate the same shape repeatedly.

Checklist:
- Provide headings in the order they should appear.
- Keep required fields obvious.
- Mark optional fields so they can be omitted cleanly.
