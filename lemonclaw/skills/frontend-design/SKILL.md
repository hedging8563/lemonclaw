# Frontend Design

This is a `generator` skill.

## Entry Rule

Use this skill when the user wants a designed frontend outcome, not just a small style fix.

## Runtime Boundary

- Skill owns: visual direction, composition, hierarchy, and implementation style.
- Runtime owns: task sequencing, approvals, and deployment flow.

## Workflow

1. Pick a strong visual direction.
2. Map the layout around the actual user goal.
3. Implement the UI in working code.
4. Verify responsiveness and polish.

## Design Rules

- Commit to a specific aesthetic, not a generic default.
- Use expressive typography and a clear palette.
- Prefer one memorable motion idea over many weak micro-animations.
- Break predictable layouts when it helps the story.
- Keep the result usable on desktop and mobile.

## Avoid

- generic purple-on-white SaaS styling
- interchangeable card grids with no hierarchy
- timid typography
- visual effects with no purpose

## Verify Before Delivering

- clear hierarchy
- mobile and desktop both work
- buttons and primary actions are obvious
- the result looks intentional, not boilerplate
