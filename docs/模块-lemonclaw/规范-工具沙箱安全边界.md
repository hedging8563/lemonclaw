# 工具沙箱安全边界 (home_dir)

> 变更日期: 2026-03-08

## 背景

LemonClaw 的 `restrict_to_workspace=true` 限制 AI agent 只能访问特定目录。原始设计中，安全边界 = 工作目录 (`~/.lemonclaw/workspace/`)，导致 agent 无法访问同级目录（如 `media/`、`config.json`、`skills/`）。

## 设计

引入 `home_dir` 概念，将安全边界从 `workspace` 扩展到其父目录 `~/.lemonclaw/`：

```
~/.lemonclaw/           ← home_dir (安全边界)
├── workspace/          ← workspace (工作目录 cwd)
├── media/              ← 媒体文件 (下载的视频/音频等)
├── config.json         ← 配置文件
├── skills/             ← 技能目录
├── memory/             ← 记忆存储
└── mcp/                ← MCP 服务器数据
```

- `workspace` 仍然是所有工具的 cwd（工作目录）
- `home_dir = workspace.parent` 是安全边界，决定哪些路径可达
- `restrict_to_workspace=true` 的语义不变，config-sync v6 仍强制开启

## 受影响的组件

| 文件 | 改动 |
|------|------|
| `agent/loop.py` | `allowed_dir=home_dir`，传 `home_dir` 给 ExecTool/CodingTool/BrowserTool |
| `agent/subagent.py` | 同 loop.py |
| `agent/tools/shell.py` | 新增 `home_dir` 参数，`_resolve_cwd` 和 `_guard_command` 用 `home_dir` 做边界检查 |
| `agent/tools/coding.py` | 新增 `home_dir` 参数，boundary 检查用 `home_dir` |
| `agent/tools/browser.py` | 新增 `home_dir` 参数，`_validate_workspace_path` 用 `home_dir` 做边界 |
| `config/schema.py` | 注释更新 |

## 未改动的组件

| 文件 | 原因 |
|------|------|
| `agent/tools/mcp.py` | MCP server 进程的 cwd 用 workspace 即可，不涉及路径校验 |
| `agent/tools/filesystem.py` | 已有 `allowed_dir` 参数，由 loop.py 传入 `home_dir` |
| `config/sync.py` | 仍强制 `restrict_to_workspace=True`，无需改动 |
| `gateway/webui/settings.py` | WebUI toggle 控制的是开关本身，语义不变 |

## 安全说明

- 边界从 `~/.lemonclaw/workspace/` 扩展到 `~/.lemonclaw/`，仍然是受限沙箱
- Agent 不能访问 `~/.lemonclaw/` 之外的任何路径
- 路径遍历检测 (`..`) 仍然生效
- 绝对路径检查仍然生效，只是边界变大了
